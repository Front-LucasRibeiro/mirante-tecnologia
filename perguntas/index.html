1. Qual é a diferença entre null e undefined em JavaScript?

    R: undefined é um valor primitivo que é atribuido a uma variável quando ela é inicializada sem valor, ou quando ela é retornada esta sem nenhum valor definido, já o tipo null é indicado para indicar a ausencia de valor, gerando em banco de dados, ou outros meios de persistência o padrão default vem como null, ou seja ausência de valor.

  2. Qual é a diferença entre classes e IDs em CSS?

    R: ID é um prop do html para selecionar apenas um elemento, usado com maior frequência para indicar elementos únicos em uma página html através do javascript, já classes em css são utilizadas para aplicar a mesma regra de estilos em um ou vários elementos.

  3. Como são atribuídas as propriedades dos objetos?

    R: São atribuidas através de uma chave e valor, o objeto é uma variável que recebe um objeto atráves chaves e suas props, na seguinte sintaxe: let objeto = { chave: "valor", nome: "Lucas" }

  4. O que são objetos de Tela?

    R: São os elementos do documento HTML, podendo ser parágrafos, imagens, botões, links entre outros.

  5. O que é DOM em JavaScript? Como usar DOM e eventos?

    R: É uma interface virtual que representa os elementos html em memória, podemos usar o DOM atribuindo escutadores de eventos aos elementos, como por exemplo, eventos de click, eventos de scroll, entre outros.

  6. Qual é o funcionamento dos temporizadores em JavaScript?

    R: Temos o setTimeOut que tem o papel de aplicar um delay(atraso) na executação de alguma tarefa no código executada uma única vez, e temos o setInterval que a cada período de tempo definido fica executando aquela tarefa que lhe foi atribuido.

  7. Qual é a diferença entre ViewState e SessionState?

    R: ViewState é usado para armazenar os estados dos elementos ou dados em uma aplicação, e o SessionState é uma funcionalidade em que podemos armazenar os dados em uma sessão do usuário.
